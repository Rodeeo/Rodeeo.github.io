try , catch , and finally -
try block. This will run the code inside the block as normal -> catch block
catch block will only run if an exception is thrown inside the try block. -> finally block
This will always be executed after the try or catch block, regardless of whether an exception occurred or not. 
It is useful if you want some code to run in both cases.

Test-driven development(TDD) is the process of writing tests before any actual code.(good luck getting someone to pay for this)

the controller pattern

the controller pattern MV* 
    MVC model view controller: model, controller, view
    Model - the brain - respoinsible for getting data: (SELECT, INSERT, UPDATE, DELETE)
    View - the html/css - communicates with the controller. can get dynamic values from the controller
    Controller - gets user input. processes requests (GET, POST, PUT, DELETE) gets data from the model. 
    passes data to the view. 
    Model = where to save the data 
    View = displaying data 
    Controller = where to message the data before you display
